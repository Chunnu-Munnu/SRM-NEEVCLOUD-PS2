# DemoGen

DemoGen is an AI-powered interactive demo generation system for NeevCloud.

It turns a real product workflow into a shareable walkthrough by combining:
- intent-driven flow planning
- whole-session screen capture
- AI-based step selection after recording
- contextual narration generation
- persona-aware explanations
- multilingual narration text
- interactive playback with highlight overlays, voice, API context, and export

## What The Project Does

A user gives DemoGen a natural-language intent such as:

`Show me how to launch a GPU instance for ML training on NeevCloud.`

DemoGen then:
1. Plans an ordered NeevCloud flow for that goal.
2. Lets the user record a real NeevCloud browser session.
3. Captures distinct moments from the full session in the background.
4. Reviews the entire capture set after recording stops.
5. Selects the most relevant 4-7 steps for the final story.
6. Generates narration, highlight boxes, API mappings, and code context.
7. Builds an interactive walkthrough that can be played, localized, exported, and shared.

## Why DemoGen Matters For NeevCloud

NeevCloud can use DemoGen for:
- onboarding new users faster
- creating reusable product demos for sales and support
- explaining GPU workflows more clearly to developers
- generating persona-specific walkthroughs from one captured flow
- localizing demos for broader adoption
- reducing the effort needed to re-record or re-explain the same product flow repeatedly

## What Makes DemoGen Different

Most teams build recorders.
DemoGen focuses on understanding and storytelling.

Key differentiators:
- Persona-aware narration: one flow, different explanations for new users, developers, and enterprise admins
- Whole-session capture review: the system records the full journey first, then decides which moments matter most
- Interactive output: the result is not just a video, but a clickable walkthrough
- Product-aware explanations: DemoGen explains why each step matters, not just what appears on screen
- API and code context: walkthrough steps can include corresponding API calls and code snippets
- Multilingual experience: English, Hindi, Tamil, and Kannada narration text

## Current Product Flow

DemoGen currently works best in this mode:
- the user opens the real NeevCloud portal in a browser
- the user clicks `Start Recording` in DemoGen
- the user shares the NeevCloud tab or window
- the user navigates the full flow normally
- DemoGen records distinct moments in the background
- the user clicks `Finish Recording`
- DemoGen reviews the full captured session and chooses the most relevant moments for the final walkthrough

This approach is intentionally practical for protected portals where direct browser automation may be unreliable.

## How To Run DemoGen

### Recommended NeevCloud Flow

1. Open DemoGen in the browser.
2. Enter a NeevCloud intent.
3. Choose a persona.
4. Choose a language.
5. Click `Start Recording`.
6. Share the NeevCloud tab.
7. Navigate through the full NeevCloud workflow normally.
8. Pause briefly on important screens.
9. Click `Finish Recording`.
10. Let DemoGen review the full recording and generate the walkthrough.

### Good Prompt To Use

```text
Show me how to launch a GPU instance for ML training on NeevCloud: open dashboard, go to GPU Instances, click Launch New Instance, choose region, select GPU type, pick image, add SSH key, review estimated pricing, and reach the final launch confirmation step.
```

## Demo Tips

For the best output:
- record 20-30 seconds of meaningful navigation
- pause 3-4 seconds on key screens
- cover 5-7 important moments
- avoid rapid back-and-forth transitions
- use `Capture Extra Moment` if there is a specific screen you definitely want included

## What Happens After Recording

When recording ends, DemoGen:
1. collects all stored capture moments
2. removes near-duplicate frames
3. selects the most relevant moments for the flow
4. maps them to the planned NeevCloud journey
5. generates step narration and multilingual text
6. builds the final interactive walkthrough

## Troubleshooting

### 1. Browser Capture Does Not Start
Make sure you are using a browser that supports `getDisplayMedia()`.
Recommended:
- Chrome
- Edge

### 2. No Steps Are Generated
Possible reasons:
- recording was too short
- screens changed too quickly
- no distinct moments were captured

Fix:
- record again for a little longer
- stay on important pages for 3-4 seconds
- use `Capture Extra Moment` on important pages

### 3. Voice Sounds Like English Even In Another Language
This usually means the browser or OS does not have a native Hindi/Tamil/Kannada voice installed.
The translated text is still correct, but the speech engine may fall back to the nearest available voice.

### 4. Gemini Feels Slow
This can happen with image-heavy sessions.
Ways to improve it:
- record fewer but cleaner moments
- avoid capturing too many unnecessary transitions
- keep the walkthrough focused on one NeevCloud use case

### 5. NeevCloud Blocks Direct Automation
That is why whole-session recording is the recommended path.
Instead of relying on browser automation, DemoGen records the users real session and then reasons over the full recorded flow.
